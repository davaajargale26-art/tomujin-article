# Tomujin Article

News/article website with a static HTML/CSS/JavaScript frontend and a Node.js/Express backend.

## Features

- Public homepage, category filter, search, article cards, and full article pages.
- Featured hero carousel from the database, limited to 4 articles.
- Hero slides show category, backend-generated publish date, view count, and author when available.
- Article view counts.
- Single-admin login with a 4-digit admin code.
- JWT-protected admin API routes.
- Admin dashboard for create, edit, delete, feature/unfeature, featured ordering, image upload, and hero preview.
- MySQL storage with in-memory fallback for local/demo availability.
- Image URL validation plus authenticated local image uploads to `frontend/public/images/uploads`.

## Project Structure

```text
backend/index.js                  Express API, auth, MySQL, uploads, fallback store
backend/.env.example              Safe environment template
frontend/public/index.html        Frontend templates
frontend/public/app.js            Frontend behavior
frontend/public/styles.css        Frontend styles
frontend/public/images/           Public images
```

## Admin Environment Setup

Copy `backend/.env.example` to `backend/.env` and fill in real values. Keep `backend/.env` local only.

Recommended session value:

```text
JWT_SECRET=<32+ character random secret>
```

Optional database values:

```text
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=
DB_NAME=loginapp
```

## Admin Login

The admin login flow is:

1. Open `/admin`.
2. Enter your admin email and password.
3. Open `/admin/dashboard`.

## Article Rules

- The admin chooses the category from a dropdown.
- The admin does not enter a publish date.
- New article dates are generated by the backend.
- Article cards, article pages, and hero slides use the selected category.
- Only 4 articles can be featured at once.
- Featured order is controlled in the admin dashboard.
- Uploaded images are limited to 3 MB and must be jpg, png, webp, avif, or gif.

## Database Notes

The backend creates and migrates these article fields automatically:

```text
is_featured
featured_order
view_count
image_url
```

If MySQL is unavailable, the app starts with sample in-memory articles. That fallback is for demos only: new articles disappear when the server restarts unless MySQL is working.

## Do Not Upload Secrets Or Heavy Folders

Never include these in a zip or deployment upload:

```text
backend/.env
.env
node_modules/
backend/node_modules/
.git/
```

The included `.gitignore` already ignores `.env` and `node_modules`. When making a zip, create it from clean project files only, or use `git archive` after committing safe files.
